     329             :  *
     330             :  * This is Daniel J. Bernstein's popular `times 33' hash function as
     331             :  * posted by him years ago on comp.lang.c. It basically uses a function
     332             :  * like ``hash(i) = hash(i-1) * 33 + str[i]''. This is one of the best
     333             :  * known hash functions for strings. Because it is both computed very
     334             :  * fast and distributes very well.
     335             :  *
     336             :  * The magic of number 33, i.e. why it works better than many other
     337             :  * constants, prime or not, has never been adequately explained by
     338             :  * anyone. So I try an explanation: if one experimentally tests all
     339             :  * multipliers between 1 and 256 (as RSE did now) one detects that even
     340             :  * numbers are not useable at all. The remaining 128 odd numbers
     341             :  * (except for the number 1) work more or less all equally well. They
     342             :  * all distribute in an acceptable way and this way fill a hash table
     343             :  * with an average percent of approx. 86%.
     344             :  *
     345             :  * If one compares the Chi^2 values of the variants, the number 33 not
     346             :  * even has the best value. But the number 33 and a few other equally
     347             :  * good numbers like 17, 31, 63, 127 and 129 have nevertheless a great
     348             :  * advantage to the remaining numbers in the large set of possible

     353             :  * numbers should be preferred and seems to be the reason why Daniel J.
